## Team Rules

- **3 to 5 members** per team
- **Diverse teams are strongly recommended** — a mix of developers and creative/functional members works best, especially for building demo videos and presentations
- Register your team on site at the event

---

## Submission Guidelines

- All projects must be submitted by **22:15**
- Each team creates a **folder in this repository** named: `team-<your-team-name>/`
- Your submission folder must contain:
  - `README.md` — short description of your solution and which challenge you addressed
  - Source code (or link to your own repo)
  - Your presentation or demo video (or a link)
- Presentations should be **5–10 minutes** — a pitch, elevator pitch, or demo video format is preferred

## How to Upload Your Solution

Follow these steps to submit your work to this repository.

### 1. Fork or clone this repository

```bash
git clone https://github.com/<org>/sap-genai-hackathon-2026.git
cd sap-genai-hackathon-2026
```

### 2. Create your team folder

Create a new folder at the root level using the naming convention `team-<your-team-name>`:

```bash
mkdir team-awesome-builders
cd team-awesome-builders
```

### 3. Add your files

Your folder should contain at minimum:

```
team-<your-team-name>/
├── README.md           ← describe your solution and which challenge you tackled
├── src/                ← your source code
│   └── ...
├── demo/               ← demo video, screenshots, or poster (or add a link in README)
│   └── ...
└── presentation/       ← slides if used (optional)
    └── ...
```

Your `README.md` should cover:

- **Challenge number and title** you addressed
- **Team members**
- **What you built** — a short description of the solution
- **How to run it** — setup and usage instructions
- **Tech stack** — languages, frameworks, APIs, models used
- **Link to demo video or live demo** (if hosted externally)

### 4. Commit and push

```bash
git add .
git commit -m "Add solution: Team Awesome Builders – Challenge 08"
git push origin main
```

If you do not have write access, open a **Pull Request** and a maintainer will merge it.

### 5. Deadline

All pushes and pull requests must be completed by **22:15** on hackathon day. Submissions after this time will not be considered for judging.

> **Tip:** Push early and push often — do not wait until the last minute. You can keep updating your folder right up to the deadline.

## Presentation Format

Think of it as a pitch or product demo, not a technical lecture. The goal is to communicate the **business value** of your solution clearly and quickly.

Formats that work well:

- A live demo with a short verbal pitch
- A pre-recorded demo video
- A poster or visual walkthrough

Slides are optional — if you use them, keep them minimal.

---

## Judging Criteria

All solutions will be evaluated on:

1. **Functionality** — Does it address the problem well?
2. **Degree of Innovation** — Is the approach unique and creative?
3. **Demo & Pitch** — How well is the solution presented?
4. **Storytelling** — Is the business value communicated clearly and engagingly?
5. **Technical Execution** — Code quality, architecture, and feasibility
